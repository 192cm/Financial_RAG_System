# Financial RAG System - Project Structure & File Descriptions

본 프로젝트는 금융 문서(사업보고서)를 기반으로 한 하이브리드 RAG 시스템의 성능을 평가하고 최적의 방법론을 도출하기 위해 설계되었습니다.

## 1. Core Source Code (src/)

### src/config.py
- **역할**: 프로젝트 전체의 설정을 중앙 관리합니다.
- **상세**: BASE_DIR, 데이터 경로(raw, db), 인덱스 이름 등을 정의하며 `.env` 파일 로드 및 API Key 설정을 담당합니다.

### src/models.py
- **역할**: LLM 모델(Gemini) 인스턴스를 생성하는 팩토리 모듈입니다.
- **성능 강화**: 503 에러 및 Rate Limit 대응을 위해 `max_retries(10회)`와 `timeout(120초)`이 적용되어 있습니다.

### src/__init__.py
- **역할**: 패키지 레벨의 인터페이스를 정의합니다. 주요 클래스와 함수들을 외부에서 쉽게 불러올 수 있도록 노출합니다.

---

## 2. Retrieval Modules (src/retrieval/)

### src/retrieval/vision_engine.py
- **역할**: ColPali 모델을 기반으로 한 비전 검색기(Vision-only Retrieval)를 담당합니다.
- **기능**: 인덱스 로드(`load_index`), 신규 인덱스 구축(`index`), 이미지 검색(`search`)을 수행합니다.

### src/retrieval/text_engine.py
- **역할**: 텍스트 기반의 하이브리드 검색기(BM25 + Semantic)를 담당합니다.
- **기능**: ParentDocumentRetriever를 활용하여 텍스트 청크를 검색하며, BM25와 앙상블되어 검색 효율을 높입니다.

### src/retrieval/router.py
- **역할**: 질문 분석을 통한 지능형 라우팅 및 필터링을 담당합니다.
- **기능**: `extract_target_company`, `extract_target_table`을 통해 질문의 의도를 파악하고, 기업별 필터링된 검색기(`get_company_filtered_retriever`)를 동적으로 생성합니다.

### src/retrieval/reranker.py
- **역할**: BGE-Reranker-v2-m3 모델을 이용한 로컬 기반 시맨틱 재순위화 레이어입니다.
- **기능**: 초기 검색된 후보군에 대해 질문과의 문맥 유사도 점수를 계산하여 순위를 재조정합니다.

---

## 3. RAG Engines (src/engines/)

### src/engines/rag_engines.py
- **역할**: Method 0-3의 모든 워크플로우를 통합 관리하는 메인 엔진입니다.
- **최신 고도화**: SOTA(Method 3) 모드에서 **Dual-Path Hybrid Retrieval**(텍스트+비전 병렬 검색), **Consensus Boost**(상호 검증 가중치), **Two-stage Reranking** 로직을 통합 관리합니다.
- **최적화**: 동일 기업 쿼리 가속을 위한 **retriever_cache**가 적용되어 있습니다.

### src/engines/agent.py
- **역할**: 비전을 통한 정밀 독해 및 자가 교정을 담당하는 핵심 에이전트입니다.
- **고도화**: 리트리버가 찾아낸 페이지의 **이미지를 직접 분석(2차 비전 검증)**하여 정답의 정합성을 확정합니다. `NEXT_PAGE_NEEDED`, `PREV_PAGE_NEEDED` 판단을 통해 표 단절이나 단위 누락 시 컨텍스트를 동적으로 확장합니다.

---

## 4. Evaluation System (src/evaluation/)

### src/evaluation/runner.py
- **역할**: 전체 평가 데이터셋을 순회하며 Method 0-3의 성능을 측정하는 메인 러너입니다.
- **기능**: 모든 방법론(Ablation Study 포함)을 병렬적으로(논리적으로) 실행하며 Latency, 토큰 사용량 등을 집계합니다.

### src/evaluation/judge.py
- **역할**: LLM-as-a-Judge 기법을 사용하여 생성된 답변과 정답(GT)을 비교하여 정성적 점수를 매깁니다.

### src/evaluation/metrics.py
- **역할**: Exact Match(EM), ROUGE-L, BLEU 메트릭을 계산합니다.

---

## 5. Utilities (src/utils/)

### src/utils/vision.py
- **역할**: 이미지 처리 및 PDF 페이지 추출 관련 헬퍼 함수 모음입니다.

### src/utils/common.py
- **역할**: 범용적인 텍스트 전처리 및 응답 정제 유틸리티입니다.
- **기능**: `clean_llm_response()`를 통해 LLM 응답에서 마크다운 태그 등을 제거합니다.

---

## 6. Notebooks & Data

### notebooks/
- `01_baseline_rag.ipynb`: 기본 텍스트 하이브리드 RAG 테스트
- `02_method1_vision_rag.ipynb`: ColPali 기반 비전 검색 성능 최적화 실험
- `03_method2_dual_path.ipynb`: Method 2 (Basic Dual-Path) 시연
- `04_method3_sota_rag.ipynb`: **Method 3 (SOTA Agentic RAG)** 기술 시연 및 코어 로직 분석
- `05_evaluation_results.ipynb`: **실험 결과 요약 (Benchmark Results)** 및 정량적 성능 지표 분석

### data/
- `raw/`: 원본 PDF 사업보고서 보관
- `chroma_db/`: 텍스트 벡터 인덱스 저장소
- `.byaldi/`: ColPali(Byaldi) 비전 인덱스 저장소
- `eval_dataset.json`: 질문, 정답, 관련 회사 정보가 포함된 평가용 데이터셋
- `docstore.pkl`: 원본 문서 저장소
